## ⚔️ CRU - Step 2: Drill (高压训练)

### 🧮 算法热身 (15min)

**题目**: LeetCode 15 - 3 Sum
**链接**: https://leetcode.cn/problems/3sum/
**状态**: ✅ Pass

**题目描述**:
给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请返回所有和为 0 且不重复的三元组。

**示例**:
```
输入: nums = [-1,0,1,2,-1,-4]
输出: [[-1,-1,2],[-1,0,1]]
解释:
- nums[0] + nums[2] + nums[4] = -1 + 2 + -1 = 0
- nums[1] + nums[2] + nums[3] = 0 + 1 + 2 = 1 ≠ 0
- nums[0] + nums[3] + nums[4] = -1 + 2 + -1 = 0
- nums[1] + nums[2] + nums[4] = 0 + 1 + -1 = 0
不同的三元组是 [-1, 0, 1] 和 [-1, -1, 2]
注意，输出的顺序和三元组的顺序并不重要。
```

### 📝 算法解法详解

#### 解法一：暴力枚举

**核心思想**：三层循环枚举所有可能

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Set<List<Integer>> result = new HashSet<>();
        int n = nums.length;

        for (int i = 0; i < n - 2; i++) {
            for (int j = i + 1; j < n - 1; j++) {
                for (int k = j + 1; k < n; k++) {
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        List<Integer> triplet = Arrays.asList(nums[i], nums[j], nums[k]);
                        Collections.sort(triplet);
                        result.add(triplet);
                    }
                }
            }
        }

        return new ArrayList<>(result);
    }
}
```

**复杂度分析**:
- 时间复杂度: O(n³)
- 空间复杂度: O(n)

---

#### 解法二：排序 + 双指针（推荐）

**核心思想**：先排序，固定一个数，用双指针找另外两个数

```
双指针图示 (nums = [-1,0,1,2,-1,-4] 排序后 [-4,-1,-1,0,1,2]):
┌─────────────────────────────────────────────────────────────┐
│  第1轮: i=0, nums[i]=-4, target=4                          │
│         i    left                       right              │
│        -4    -1   -1   0   1   2                            │
│                                  ↑    ↑                      │
│                              -4+1+2=-1≠0                    │
│                         两指针相向移动...                        │
└─────────────────────────────────────────────────────────────┘

去重技巧: 跳过重复元素
if (i > 0 && nums[i] == nums[i-1]) continue;
while (left < right && nums[left] == nums[left+1]) left++;
```

**代码实现**:
```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);  // 先排序

        int n = nums.length;

        for (int i = 0; i < n - 2; i++) {
            // 去重: 跳过重复的第一个数
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue;
            }

            int left = i + 1;
            int right = n - 1;
            int target = -nums[i];

            while (left < right) {
                int sum = nums[left] + nums[right];

                if (sum == target) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // 去重: 跳过重复的第二个数和第三个数
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    while (left < right && nums[right] == nums[right - 1]) right--;

                    left++;
                    right--;
                } else if (sum < target) {
                    left++;  // 和太小，左指针右移
                } else {
                    right--;  // 和太大，右指针左移
                }
            }
        }

        return result;
    }
}
```

**复杂度分析**:
- 时间复杂度: O(n²) - 排序 O(n log n) + 双指针 O(n²)
- 空间复杂度: O(1) - 不考虑返回结果

---

### 🔍 两种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| **暴力枚举** | O(n³) | O(n) | 思路简单 | 效率太低 |
| **双指针** | O(n²) | O(1) | 效率高，去重简单 | 需要排序 |

### ⚠️ 常见错误

1. **忘记排序**
```java
// ❌ 错误代码
public List<List<Integer>> threeSum(int[] nums) {
    // 没有 Arrays.sort(nums)
    // 双指针法的前提是数组有序！
}
```

2. **去重不完整**
```java
// ❌ 错误代码
for (int i = 0; i < n - 2; i++) {
    // 忘记跳过重复的 nums[i]
    // 导致重复的三元组
}
```

### 2.2 put() 流程

```java
public V put(K key, V value) {
    Segment<K,V> s = segmentForHash(hash);  // 1. 定位 Segment
    s.lock();                                // 2. 锁住这个 Segment
    try {
        // 3. 在 Segment 内部操作 HashEntry[]
        HashEntry<K,V>[] tab = s.table;
        int index = (tab.length - 1) & hash;
        HashEntry<K,V> e = entryAt(tab, index);

        // 遍历链表，查找或插入
        for (;;) {
            if (e == null) {
                // 插入新节点（头插法）
                entryAt(tab, index).set(new HashEntry<K,V>(key, value));
                break;
            }
            if (e.key == key) {
                // 更新已有节点
                e.value = value;
                break;
            }
            e = e.next;
        }
    } finally {
        s.unlock();  // 4. 释放锁
    }
}
```

### 3.2 put() 流程

```java
final V putVal(K key, V value, boolean onlyIfAbsent) {
    if (key == null || value == null) throw new NullPointerException();

    int hash = spread(key.hashCode());  // 扰动函数

    for (Node<K,V>[] tab = table;;) {
        Node<K,V> f; int n, i, fh;

        if (tab == null || (n = tab.length) == 0)
            tab = initTable();  // 初始化数组

        else if ((f = tabAt(tab, i = (n - 1) & hash)) == null) {
            // 🔥 关键点1: CAS 无锁插入
            if (casTabAt(tab, i, null, new Node<K,V>(hash, key, value)))
                break;  // 成功，直接退出
        }

        else if ((fh = f.hash) == MOVED)
            tab = helpTransfer(tab, f);  // 帮助扩容

        else {
            // 🔥 关键点2: synchronized 锁节点
            synchronized (f) {  // 只锁住链表/树的头节点
                // 在链表或红黑树中插入或更新
            }
        }
    }
}

│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  1. Node 的 val 和 next 都用 volatile 修饰                            │   │
│  │  2. volatile 保证可见性:                                            │   │

│  ┌─────────────────────────────────────────────────────────────────────┐   │

│  │    - 写操作: 立即刷新到主内存                                        │   │

## 六、扩容机制

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ConcurrentHashMap 扩容机制                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【触发条件】                                                                │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  当 map.size() > 负载因子 × 数组长度时，触发扩容                       │   │
│  │                                                                     │   │
│  │  负载因子 = 0.75 (与 HashMap 相同)                                    │   │
│  │                                                                     │   │
│  │  例: 容量 16，存入 12 个元素 → 扩容到 32                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【JDK 1.7 扩容】                                                            │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  1. 创建新的 Segment 数组（容量翻倍）                                 │   │
│  │  2. 重新计算每个元素在新数组中的位置                                   │   │
│  │  3. 迁移元素                                                         │   │
│  │                                                                     │   │
│  │  问题: 扩容期间需要持有所有 Segment 的锁，并发度降低                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【JDK 1.8 扩容（多线程协同）】                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  1. 创建新数组（容量翻倍）                                            │   │
│  │  2. 设置 sizeCtl 为负数（标记正在扩容）                                │   │
│  │  3. 其他线程协助扩容:                                               │   │
│  │     - 每个线程负责迁移一小部分                                        │   │
│  │     - 迁移完成后继续 put 操作                                         │   │
│  │                                                                     │   │
│  │  优势: 多线程并行扩容，扩容期间仍可 put（不冲突的部分）                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  【ForwardingNode】                                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  扩容时，旧数组的位置会放置一个 ForwardingNode                         │   │
│  │                                                                     │   │
│  │  作用:                                                              │   │
│  │  - 标记该位置已迁移到新数组                                           │   │
│  │  - 指向新数组的位置                                                 │   │
│  │  - 其他线程看到 ForwardingNode，知道数组正在扩容                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

## 七、常见面试题

### Q1: ConcurrentHashMap 1.7 vs 1.8 本质区别？

**标准答案**：
1. **数据结构**: 1.7 Segment + HashEntry，1.8 Node + TreeNode
2. **锁机制**: 1.7 ReentrantLock，1.8 synchronized
3. **锁粒度**: 1.7 Segment 级别，1.8 Node 级别
4. **无锁操作**: 1.7 无，1.8 有（CAS）

### Q2: ConcurrentHashMap 的 get() 为什么不加锁？

**标准答案**：
- Node 的 `val` 和 `next` 字段都用 `volatile` 修饰
- volatile 保证可见性：写操作立即刷新到主内存，读操作从主内存读取
- get() 只是读取，不修改数据结构，无需加锁

### Q3: ConcurrentHashMap 在高并发下如何保证线程安全？

**标准答案**：
- **JDK 1.7**: Segment 分段锁，不同 Segment 可以并发
- **JDK 1.8**: CAS + synchronized，首节点无锁插入，后续竞争加锁
- **扩容**: 多线程协同扩容，提高效率

### Q4: synchronized 为什么比 ReentrantLock 快？

**标准答案**：
- JDK 1.6 后 synchronized 做了大量优化（锁升级、偏向锁、轻量级锁）
- 在竞争不激烈的情况下，性能接近 ReentrantLock
- synchronized 使用更方便，编译器优化空间更大

## 八、常见误区

### ❌ 误区1：ConcurrentHashMap 完全无锁

```
正确理解：
- JDK 1.8 首节点插入时用 CAS（无锁）
- 但有竞争时仍需用 synchronized（有锁）
- 不是完全无锁，是"低锁开销"
```

### ❌ 误区2：ConcurrentHashMap 可以保证复合操作原子性

```
正确理解：
ConcurrentHashMap 只保证单个方法的线程安全：
- put() 线程安全
- get() 线程安全

但复合操作不安全：
if (!map.containsKey(key)) {  // ← 线程A检查
    map.put(key, value);       // ← 线程B可能先插入
}

解决方案：使用 putIfAbsent() 或 ConcurrentHashMap 的原子方法
```

### ❌ 误区3：ConcurrentHashMap 弱一致性问题

```
正确理解：
ConcurrentHashMap 的 get() 返回最新值（volatile 保证）
不是弱一致性，而是强一致性
```

### 9.1 grep - 文本搜索

```bash
# 基本用法
grep "关键字" file.txt           # 搜索关键字

# 常用选项

grep -r "import" ./src           # 递归搜索目录

# 正则表达式
grep "^[0-9]" file.txt          # 以数字开头的行
grep "hello.*world" file.txt    # hello 后面跟 world
```

### 9.2 awk - 文本处理

```bash
# 基本用法
awk '{print $1}' file.txt        # 打印第1列
awk '{print $1, $NF}' file.txt   # 第1列和最后一列（$NF是内置变量）

# 指定分隔符
awk -F: '{print $1}' /etc/passwd # 以冒号分隔，打印第1列
awk -F, '{print $2}' data.csv   # 以逗号分隔，打印第2列

# 条件过滤
awk '$3 > 100 {print $0}' file.txt # 第3列大于100的行
awk '/ERROR/ {print $0}' log.txt   # 包含 ERROR 的行

# 统计
awk '{sum+=$1} END {print sum}' file.txt  # 统计第1列总和
awk 'END {print NR}' file.txt              # 统计总行数
```

### 9.3 sed - 文本编辑

```bash
# 基本用法
sed 's/旧/新/g' file.txt         # 替换（不修改原文件，输出到屏幕）
sed 's/DEBUG/INFO/g' log.txt    # 把所有 DEBUG 替换为 INFO

# 直接修改文件（谨慎使用）
sed -i 's/DEBUG/INFO/g' log.txt  # 直接修改文件

# 删除行
sed '/ERROR/d' log.txt          # 删除包含 ERROR 的行
sed '3d' file.txt               # 删除第3行

# 指定行操作
sed '3s/old/new/' file.txt      # 只替换第3行
sed '1,5s/old/new/' file.txt    # 替换1到5行
```

### 9.4 组合使用

```bash
# 查找并统计
grep "ERROR" log.txt | awk '{print $NF}' | sort | uniq -c

# 查找并替换
grep -r "TODO" ./src | sed 's/TODO/FIXME/g'

# 查找并删除
grep -v "注释" file.txt | sed '/^$/d'
```
