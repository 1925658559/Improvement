### 🧮 算法热身 (15min)

**题目**: LeetCode 21 - Merge Two Sorted Lists
**链接**: https://leetcode.cn/problems/merge-two-sorted-lists/
**状态**: ✅ Pass

**题目描述**:
将两个升序链表合并为一个新的 **升序** 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

**示例**:
```
输入: list1 = [1,2,4], list2 = [1,3,4]
输出: [1,1,2,3,4,4]
```

---

### 📝 算法解法详解

#### 解法一：迭代法（推荐）

**核心思想**：使用 dummy 哨兵节点，双指针比较两个链表，较小的节点接入新链表。

```
list1: 1 → 2 → 4 → null
list2: 1 → 3 → 4 → null

过程演示：
┌─────────────────────────────────────────────────────────────┐
│  轮次 │   dummy 链表       │   p1(list1)  │   p2(list2)     │
├─────────────────────────────────────────────────────────────┤
│  初始 │ dummy → null       │   1          │   1             │
│  第1轮│ dummy → 1 → null   │   2          │   1             │
│  第2轮│ dummy → 1 → 1 → null│  2          │   3             │
│  第3轮│ dummy → 1 → 1 → 2 → null│ 4      │   3             │
│  第4轮│ dummy → 1 → 1 → 2 → 3 → null│ 4   │   4            │
│  第5轮│ dummy → 1 → 1 → 2 → 3 → 4 → null│ null │ 4        │
│  第6轮│ dummy → 1 → 1 → 2 → 3 → 4 → 4 → null│ null │ null │
└─────────────────────────────────────────────────────────────┘

返回: dummy.next
```

**代码实现**:
```java
/**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        // 哨兵节点：简化边界处理
        ListNode dummy = new ListNode(0);
        ListNode cur = dummy;

        ListNode p1 = list1;
        ListNode p2 = list2;

        // 双指针比较，较小的接入新链表
        while (p1 != null && p2 != null) {
            if (p1.val <= p2.val) {
                cur.next = p1;
                p1 = p1.next;
            } else {
                cur.next = p2;
                p2 = p2.next;
            }
            cur = cur.next;
        }

        // 处理剩余节点（只有一个链表还有节点）
        cur.next = (p1 != null) ? p1 : p2;

        return dummy.next;
    }
}
```

**复杂度分析**:
- 时间复杂度: O(n + m) - n 和 m 分别是两个链表的长度
- 空间复杂度: O(1) - 只用了常数级别的额外空间

---

#### 解法二：递归法

**核心思想**：每次比较两个链表的头节点，较小的节点作为合并后链表的头，然后递归合并剩余部分。

```
mergeTwoLists([1,2,4], [1,3,4])
├─ 1 <= 1 → 返回 1，递归 merge([2,4], [1,3,4])
│  └─ 2 > 1 → 返回 1，递归 merge([2,4], [3,4])
│     └─ 2 < 3 → 返回 2，递归 merge([4], [3,4])
│        └─ 4 > 3 → 返回 3，递归 merge([4], [4])
│           └─ 4 <= 4 → 返回 4，递归 merge([], [4])
│              └─ list1 为空 → 返回 [4]
```

**代码实现**:
```java
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        // Base case: 有一个链表为空
        if (list1 == null) {
            return list2;
        }
        if (list2 == null) {
            return list1;
        }

        // 比较头节点，较小的作为合并后的头
        if (list1.val <= list2.val) {
            list1.next = mergeTwoLists(list1.next, list2);
            return list1;
        } else {
            list2.next = mergeTwoLists(list1, list2.next);
            return list2;
        }
    }
}
```

**复杂度分析**:
- 时间复杂度: O(n + m)
- 空间复杂度: O(n + m) - 递归栈深度

---

### 🔍 三种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|------|------------|------------|------|------|
| **迭代法** | O(n + m) | O(1) | 无栈溢出风险，生产推荐 | 代码稍长 |
| **递归法** | O(n + m) | O(n + m) | 代码简洁优雅 | 栈溢出风险 |

### ⚠️ 常见错误

1. **忘记处理剩余节点**
```java
// ❌ 错误代码
while (p1 != null && p2 != null) {
    // ...
}
// 缺少：cur.next = (p1 != null) ? p1 : p2;
return dummy.next;
```

2. **cur 指针移动错误**
```java
// ❌ 错误代码
if (p1.val <= p2.val) {
    cur.next = p1;
    // 缺少：p1 = p1.next;
}
cur = cur.next;  // cur 会指向原来的 p1.next，导致死循环
```

3. **递归没有返回 list1/list2**
```java
// ❌ 错误代码
if (list1.val <= list2.val) {
    list1.next = mergeTwoLists(list1.next, list2);
    // 缺少：return list1;
}
```

### 1.5 面试题

#### Q1: 线程池的 7 个参数分别是什么？

**标准答案**：
```java
ThreadPoolExecutor(
    int corePoolSize,           // 核心线程数：始终保持存活的线程
    int maximumPoolSize,        // 最大线程数：线程池允许的最大线程数
    long keepAliveTime,         // 空闲线程存活时间：非核心线程空闲超过该时间被回收
    TimeUnit unit,              // 存活时间单位
    BlockingQueue<Runnable> workQueue,  // 任务队列：存放等待执行的任务
    ThreadFactory threadFactory,  // 线程工厂：创建新线程的工厂
    RejectedExecutionHandler handler  // 拒绝策略：队列满且线程数达到最大时的处理策略
)
```

#### Q2: 线程池的工作流程是什么？

**标准答案**：
1. 如果当前线程数 < corePoolSize，创建核心线程执行任务
2. 如果当前线程数 ≥ corePoolSize，任务放入队列等待
3. 如果队列已满且当前线程数 < maximumPoolSize，创建非核心线程执行任务
4. 如果队列已满且当前线程数 ≥ maximumPoolSize，执行拒绝策略

#### Q3: 为什么不推荐使用 Executors 创建线程池？

**标准答案**：
- **FixedThreadPool/SingleThreadExecutor**: 使用 `LinkedBlockingQueue` 无界队列，任务堆积可能导致 OOM
- **CachedThreadPool**: 线程数无上限（Integer.MAX_VALUE），可能创建大量线程导致 OOM

**正确做法**：使用 `ThreadPoolExecutor` 构造函数，明确指定参数，使用有界队列。

### 2.3 AQS acquire 获取锁流程 (以 ReentrantLock 为例)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    ReentrantLock.lock() 流程                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【场景】 线程 A 已持有锁，线程 B 尝试获取锁                                    │
│                                                                             │
│  线程 B 调用 lock.lock()                                                     │
│         ↓                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  调用 AQS.acquire(1)                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│         ↓                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  tryAcquire(1)  ← 子类实现（Sync.nonfairTryAcquire）                  │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐ │   │
│  │  │  if (state == 0) {  // 无锁                                       │ │   │
│  │  │      if (CAS设置state=1成功) {                                   │ │   │
│  │  │          setExclusiveOwnerThread(current);  // 获取锁成功         │ │   │
│  │  │          return true;                                            │ │   │
│  │  │      }                                                           │ │   │
│  │  │  } else if (当前线程 == 持有锁线程) {  // 重入                     │ │   │
│  │  │      state++;  // 重入次数+1                                       │ │   │
│  │  │      return true;                                                │ │   │
│  │  │  }                                                               │ │   │
│  │  │  return false;  // 获取失败                                       │ │   │
│  │  └─────────────────────────────────────────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│         │ 失败                                                              │
│         ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  addWaiter(Node.EXCLUSIVE)  ← 将当前线程加入 CLH 队列尾部              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│         ↓                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  acquireQueued(node, 1)  ← 在队列中自旋等待                           │   │
│  │  ┌─────────────────────────────────────────────────────────────────┐ │   │
│  │  │  for (;;) {  // 自旋                                             │ │   │
│  │  │      // 前驱节点是 head，再次尝试获取锁                            │ │   │
│  │  │      if (node.prev == head && tryAcquire(1)) {                  │ │   │
│  │  │          setHead(node);  // 设置自己为 head                       │ │   │
│  │  │          return;                                                 │ │   │
│  │  │      }                                                           │ │   │
│  │  │                                                                 │ │   │
│  │  │      // 检查是否应该 park                                         │ │   │
│  │  │      if (shouldParkAfterFailedAcquire(node.prev)) {             │ │   │
│  │  │          LockSupport.park(this);  // 阻塞当前线程                 │ │   │
│  │  │      }                                                           │ │   │
│  │  │  }                                                               │ │   │
│  │  └─────────────────────────────────────────────────────────────────┘ │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.6 面试题

#### Q1: 什么是 AQS？

**标准答案**：
AQS (`AbstractQueuedSynchronizer`) 是 JUC 包中大多数同步器的底层实现基础。它使用一个 `volatile int state` 作为同步状态，一个 CLH 队列管理等待线程。

**核心思想**：模板方法模式。AQS 定义了同步器的骨架（排队、阻塞、唤醒），子类只需实现状态管理的逻辑（`tryAcquire`、`tryRelease`）。

#### Q2: AQS 的核心组件是什么？

**标准答案**：
1. **state**: 同步状态（volatile int），通过 CAS 修改
2. **CLH 队列**: 双向链表，存储等待获取锁的线程
3. **模板方法**: `acquire`/`release` 流程，子类实现 `tryAcquire`/`tryRelease`

#### Q3: AQS 的 acquire 流程是怎样的？

**标准答案**：
1. 调用 `tryAcquire` 尝试获取锁（由子类实现）
2. 如果失败，调用 `addWaiter` 将当前线程加入 CLH 队列尾部
3. 调用 `acquireQueued` 在队列中自旋等待
4. 如果前驱节点是 head，再次尝试获取锁
5. 如果获取失败，检查是否应该 `park`（阻塞）

#### Q4: ReentrantLock 是如何实现可重入的？

**标准答案**：
在 `tryAcquire` 中，首先检查 `state == 0`（无锁），如果为 0 则 CAS 设置为 1。如果 `state != 0`，检查当前线程是否是持有锁的线程，如果是则 `state++`（重入次数+1）。

在 `tryRelease` 中，`state--`，只有当 `state == 0` 时才完全释放锁。
